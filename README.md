# World News — Continual-Internalization Dataset

A Polymarket-spike-anchored news corpus for evaluating continual learning's effect on a model's
**internalization** of new world facts. Articles span Feb 18 – Mar 20, 2026 (post most LLM training
cutoffs); each was scraped because a real-money prediction market moved sharply around it.

All token counts in this doc use the `Qwen/Qwen3-30B-A3B-Instruct-2507` tokenizer.

---

## Top-level layout

```
/data/world-news/
  articles/<hash>/                      # per-article content + per-article QA
    article.txt                         # cleaned body
    raw.txt                             # raw scrape (deeper nav/cruft retained)
    metadata.json                       # {hash, sources[], appearances[]}
    train-simple-qa/qa.jsonl            # ← TRAINING questions
    direct/qa.jsonl                     # eval — Yes/No factual recall
    temporal/qa.jsonl                   # eval — 4-axis temporal-update probes
    temporal/qa_source.jsonl            # source 0-hop probes used to derive temporal/qa.jsonl
    compositional/2-hop.jsonl           # eval — 2-hop multihop yes/no
    compositional/3-hop.jsonl           # eval — 3-hop multihop yes/no
    compositional/4-hop.jsonl           # eval — 4-hop multihop yes/no

  markets/<mid>/                        # Polymarket market metadata
    metadata.json                       # market question, description, tags, volume
    spike_<sid>/
      metadata.json                     # spike timestamps, prices, articles[]
      indexing/qa.jsonl                 # per-spike "what major X in Y" questions

  indexing/                             # consolidated indexing eval
    raw.jsonl                           # 959 per-spike rows (flat)
    filtered.jsonl                      # 82 canonical rows (one per dedup'd event cluster)
    deduped.json                        # cluster metadata + members for drill-down

  boundary_abstention/                  # answer-is-"No, no record" probes
    raw.jsonl                           # 100 candidates (with retrieval+judge metadata)
    filtered.jsonl                      # 83 with n_affirms=0 in top-10 retrieval

  eval_sample/                          # the 868-row sampled eval set (see below)
    all.jsonl
    direct.jsonl, temporal.jsonl,
    compositional_{2,3,4}hop.jsonl,
    indexing.jsonl, boundary_abstention.jsonl

  valid_hashes.txt                      # 678 article hashes that constitute the corpus
  invalidated_hashes.txt                # 795 hashes excluded from the corpus
  _corpus_embeddings.npy                # cached BGE article embeddings (1473×1024)
  _corpus_embeddings_meta.json
```

## Article corpus (the unit of training)

**1,473** unique-content articles were scraped. We restrict the **valid corpus** to **678 articles**
that satisfy at least one of:

1. Length is in the **500–1500 cleaned-token band** (the modal range, well-formed mainstream news), OR
2. The article is anchored by the **51-article temporal/multihop cohort** (the curated subset whose
   article-introduced fact updates power the temporal and compositional evals).

The remaining 795 articles are listed in `invalidated_hashes.txt` (mostly ≤100-token scraping stubs
or very-long outliers). All scripts that load corpus content should filter against `valid_hashes.txt`.

| | full set | **valid corpus** | invalidated |
|---|---:|---:|---:|
| articles | 1,473 | **678** | 795 |
| cleaned tokens (Qwen3) | 1,528,121 | **638,828** | 889,293 |
| avg / article | 1,037 | 942 | 1,118 |

Heavy-tailed token distribution; in the valid set, median is 753, p90 is 1,329.

### Source provenance

Articles were scraped around price spikes on Polymarket prediction markets. Each market has
significant 7-figure trading volume and saw a sharp probability move (the "spike") triggered by
news. There are 139 markets and 921 spikes total in `markets/`. An article can appear in multiple
(market, spike) contexts; `articles/<hash>/metadata.json#appearances` lists them.

## Training data (what the model learns from)

Two streams, both restricted to the **678 valid articles**:

### 1. Article corpus (teacher context)
The cleaned `article.txt` files. **638,828 tokens** across 678 articles.

### 2. Training questions: `train-simple-qa`
**16,667** factual question-answer rows generated by `Qwen/Qwen3.6-35B-A3B` (DeepInfra) with a
self-contained, date-anchored prompt — answers are 1-2 sentence factual recalls. Coverage: 661/678
articles have ≥1 question. Avg ~25 rows per article.

| | tokens |
|---|---:|
| Q | 563,555 |
| A | 200,831 |
| **Q + A** | **764,386** |
| avg per row | 45.9 |

### Combined training token budget
- **1,403,214** total tokens (corpus 46%, Q+A 54%)
- Typical OPSD setup: teacher sees `(article, question)`, student sees `question` only and matches
  teacher logits over the answer span. So student's effective supervision signal ≈ 764K tokens of
  Q+A; teacher sees the full ~1.4M.

### Schema — `train-simple-qa/qa.jsonl`
```json
{
  "question": "...",  "answer": "...",
  "hash": "...",  "article_file": "/data/world-news/articles/<hash>/article.txt",
  "article_title": "...",  "article_published_date": "YYYY-MM-DD",
  "market_ids": [...],  "event_titles": [...]
}
```

## Evaluation data — six tasks

The default eval set is `eval_sample/all.jsonl` (**868 rows · 30,039 Q+A tokens**), sampled from
the per-article eval files with `random.seed(0)` for reproducibility.

| task | rows | sampling | what it tests |
|---|---:|---|---|
| `direct` | **180** | 60 random valid articles × 3 random Yes/No rows | factual recall (existence / boundary discrimination) |
| `temporal` | **131** | 1 random fact-update axis-set per article × 51 cohort articles | temporal-update internalization (see *axis spec* below) |
| `compositional_2hop` | **149** | 3 random per article (51 articles) | multi-hop chained reasoning, depth 2 |
| `compositional_3hop` | **150** | 3 random per article (51 articles) | multi-hop, depth 3 |
| `compositional_4hop` | **93** | all rows | multi-hop, depth 4 |
| `indexing` | **82** | all (one canonical per dedup'd event cluster) | salience-weighted ranking — does the model surface the corpus's main events when asked about a broad domain × time scope? |
| `boundary_abstention` | **83** | all surviving (n_affirms=0 in top-10 retrieval judge) | "knows what it didn't learn" — confidently asserts absence on plausible-but-uncovered claims |

### Per-task generation provenance

- **`direct`** — `Qwen/Qwen3.6-35B-A3B` (DeepInfra), 10 boolean Y/N per article, balanced ~5/5 with
  "No" claims constructed by swapping a single element of a true article fact. Generator:
  `generate_direct_qa.py`.
- **`temporal`** — derived axis-rewrite of `temporal/qa_source.jsonl` (the original 0-hop Gemini
  yes/no probes from `multihop/easy_qa_gemini.jsonl`). For each fact-update, the generator
  classifies as TYPE-A (named entity-role swap → 4 axes) or TYPE-B (state/value flip → 2 axes):
  - **T1**: forward (role → new entity / value).
  - **T2**: negation (does old entity / value still hold?). Always answer "No".
  - **T3**: prior identity preserved (TYPE-A only) — e.g., "what is Kevin Warsh otherwise known
    for?" → "Former Federal Reserve Governor (2006-2011)".
  - **T4**: backward (old entity → new entity, TYPE-A only).
  Answers are tight noun phrases. Generator: `generate_temporal_axis_qa.py`.
- **`compositional/{2,3,4}-hop`** — yes/no questions where the article introduces a fact whose
  answer requires chaining through Wikipedia-style world knowledge. Each row carries a `chain`
  field describing per-hop sources, plus `generator: "gemini"` (357 rows from
  `multihop/hard_qa_multihop_gemini_yesno.jsonl`) or `generator: "inhouse_v3"` (605 rows from
  `hard_qa_v3_multihop.jsonl`). Files are split by precise `hop_count`. Two generators on the
  same article cohort give cross-validated multihop coverage.
- **`indexing`** — generated per-spike from market + spike + article cluster context. Question is
  broad-strokes ("What major events happened in [domain × time]?"); gold is a single named-event
  rollup (e.g., "the joint U.S.-Israeli airstrikes on Iran in late February 2026"). The 959 raw
  rows were embedding-clustered (BGE-large, cosine sim ≥ 0.72) into 82 unique-event clusters; we
  use the canonical (closest-to-centroid) row from each. `cluster_size` field encodes salience.
- **`boundary_abstention`** — generated against the top-20 dedup'd indexing clusters. Each candidate
  asks a plausible follow-up across 4 strategies (actor-action absence / subevent absence /
  quote absence / quantitative-consequence absence) whose intended gold is "No, the corpus has no
  record of X". A second stage embedded each question, retrieved top-10 articles from the full
  1,473-article corpus by cosine similarity to the question, and used an LLM judge to label each
  retrieved article as `affirm` / `deny` / `unrelated`. The filtered set (83 rows) keeps only
  candidates with **n_affirms = 0** — i.e., no retrieved article supports a "Yes" answer.
  This makes them genuinely RAG-adversarial: retrieval can't find evidence to flip the answer.
- **`train-simple-qa`** (training) — `generate_easy_qa.py`. 25 self-contained QAs per article with
  explicit date anchors and full entity names.

### Source files used to build everything
The generation scripts live at the repo root (`/home/nvidia/continual-internalization/`):
- `generate_easy_qa.py` → `<hash>/train-simple-qa/qa.jsonl`
- `generate_direct_qa.py` → `<hash>/direct/qa.jsonl`
- `generate_temporal_axis_qa.py` → `<hash>/temporal/qa.jsonl` (reads `qa_source.jsonl`)
- `generate_spike_indexing_qa.py` → `markets/<mid>/spike_<sid>/indexing/qa.jsonl`
- `dedup_spike_indexing.py` → `indexing/{deduped.json, filtered.jsonl}`
- `generate_boundary_abstention.py` → `boundary_abstention/{raw,filtered}.jsonl`
- `judge_qa.py` → reusable LLM judge (`--kind direct|indexing|easy`)

All generators use DeepInfra `Qwen/Qwen3.6-35B-A3B` with `enable_thinking=False`.

## Quick-start for training

```python
import json
from pathlib import Path

WN = Path("/data/world-news")
valid = set((WN / "valid_hashes.txt").read_text().strip().split("\n"))

# (corpus_doc, question, gold_answer) tuples for OPSD/SFT
for h in valid:
    article = (WN / "articles" / h / "article.txt").read_text()
    qa_path = WN / "articles" / h / "train-simple-qa" / "qa.jsonl"
    if not qa_path.exists(): continue
    for line in qa_path.open():
        r = json.loads(line)
        yield {"corpus": article, "question": r["question"], "answer": r["answer"]}
```

For evaluation, load `eval_sample/all.jsonl` and dispatch on the `task` field.
